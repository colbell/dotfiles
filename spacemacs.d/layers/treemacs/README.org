<h1><a href="#description" aria-hidden="true" class="anchor" id="user-content-description"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Description</h1>
<p>This layer brings in <a href="https://github.com/Alexander-Miller/treemacs">Treemacs</a> to replace Neotree as the default file and project explorer.</p>
<h1><a href="#features" aria-hidden="true" class="anchor" id="user-content-features"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Features</h1>
<p>An detailed overview over Treemacs’ available features is available in <a href="https://github.com/Alexander-Miller/treemacs#detailed-feature-list">its readme</a>. In a short summary Treemacs offers
  the following:</p>
<ul>
  <li>Simple and powerful navigation and ability to detail exactly how and where a file should be opened</li>
  <li>Good looking png icons.</li>
  <li>Ability to show tags contained in files. Tags are provided by imenu, so nearly every filetype is supported.</li>
  <li>Optionally fontifying files based on their git status.</li>
  <li>Optionally collapsing single-dir-child directories into one.</li>
  <li>Doing both asynchronously for an imperceptible performance cost.</li>
  <li>Optional follow-mode to automatically focus the currently selected file.</li>
  <li>Optional filewatch-mode to automatically refresh the view after (and only after) changes to the shown filesystem.</li>
</ul>
<h1><a href="#installation" aria-hidden="true" class="anchor" id="user-content-installation"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Installation</h1>
<p>To use this layer, add <code>treemacs</code> to the existing <code>dotspacemacs-configuration-layers</code> in your <code>dotspacemacs</code> file.
  In addition you can also specify the following configuration variables:</p>
<ul>
  <li><code>treemacs-use-follow-mode</code>
    When t <code>treemacs-follow-mode</code> will be enabled.
    Default value: t</li>
  <li><code>treemacs-use-filewatch-mode</code>
    When t <code>treemacs-filewatch-mode</code> will be enabled.
    Default value: t</li>
  <li><code>treemacs-use-collapsed-directories</code>
    Maximum number of single-dir-child directories Treemacs will collapse into one.
    Default value: 3</li>
</ul>
<p>Using all 3 options the <code>dotspacemacs-configuration-layers</code> will look like this:</p>
<div class="highlight highlight-source-emacs-lisp"><pre>(treemacs
 <span class="pl-c1">:variables</span>
 treemacs-use-follow-mode <span class="pl-c1">t</span>
 treemacs-use-filewatch-mode <span class="pl-c1">t</span>
 treemacs-use-collapsed-directories <span class="pl-c1">3</span>)</pre></div>
